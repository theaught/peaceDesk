Setup

Get ready in 4 steps

1

Enter your API key

Paste your Velma2 API key into the API Key field in the top bar. The field has a placeholder — just replace it.

2

Pick a config

The FraudConfig is ready to go by default. You can also drag and drop your own .json config file onto the upload area. Only one config can be active at a time.

No config selected? The app runs in presets-only mode — minimal analysis using only the presets you choose.
3

Choose your signals

Four optional enrichments. Toggle only what you need — extras slow things down.

Speaker labels
Tells speakers apart
On
Emotion

Off
Accent

Off
Deepfake

Off
4

Add presets (optional)

Presets are extra behavior detectors. Check the ones you want, then click the refresh icon (↻) if the list doesn't load.

During a call

Running a session

▶

Start

Click the microphone button. Allow mic access when prompted. The status dot turns green and the call timer begins. The mic button turns red while recording.

⏸

End the call

Click End Call in the call header. The app sends a final signal to the API and waits for any remaining results (like a summary) before closing.

⟳

Reset between runs

The Clear button wipes the transcript and alerts panel without ending an active session.

Understanding results

Alert card colors

Conversation type Behavior flagged Topic detected Sentiment Summary Emotion ⚠ High-risk

High-risk behaviors — like vishing or account impersonation — appear in red with a ⚠ warning. Confidence scores show as a percentage bar where available.

Need help?

Common issues

What you're seeing What to try
Mic button does nothing You're opening the file directly — use the local server command above instead
Presets say "Failed to load" Your API key is missing or incorrect
Transcript feels slow Normal — the API waits for natural pauses. Turn off Speaker Labels to speed it up.
Conversation type shows as a long code The API returned a type not in your config — common in presets-only mode
No results after stopping Check the Event Log at the bottom of the right panel for connection errors

Sharing the app

Send the .html file directly — it's fully self-contained. Recipients need:

  • ✓ Their own API key (they enter it in the API Key field)
  • ✓ Chrome or Edge (recommended)
  • ✓ To open via a local server or HTTPS — not by double-clicking the file
